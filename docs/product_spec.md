# 戒烟有数产品规格

## 0. 原型、产品规格、实施计划的优先级

后续 Flutter 版与 Harmony 版实现同时参考“修改后的原型文件”和 `docs/product_spec.md`。二者不冲突时都要遵循；如果出现冲突，按以下优先级执行：

1. `docs/product_spec.md`
   - 作为最终产品规则源。
   - 决定功能范围、公式口径、交互规则、通知策略、合规提示、验收标准。

2. 修改后的原型文件
   - 作为 UI、布局、页面流程、视觉风格和交互细节参考。
   - 负责回答“长什么样、页面怎么流转、移动端密度如何”。

3. 实施计划
   - 作为工程执行拆分。
   - 负责回答“按什么阶段开发、每阶段验证什么、报告怎么输出”。

解释：

- 原型负责“长什么样、怎么流转”。
- 产品规格负责“规则是什么、哪些必须做、哪些不能做”。
- 实施计划负责“怎么分阶段落地”。

冲突处理示例：

- 如果原型里有乱码文案，不照搬，按产品规格改成可读中文。
- 如果原型里有手表端页面，Harmony 手机端首版不实现完整手表端，只保留未来入口/说明；Harmony 手表端未来单独评估。
- 如果原型或旧计划残留“严厉危害警示阻止取烟”，以产品规格为准：不阻止取烟，提交成功后只通过手机系统通知栏三连提醒。
- 如果原型与规格都没有覆盖某个 UI 细节，优先沿用原型现有设计语言，但用 ArkTS / ArkUI 原生方式实现，不照搬 Flutter 代码结构。
- 如果实施计划与产品规格冲突，先修正实施计划，再实现。

## 1. 产品定位

戒烟有数是一款以“看见库存、看见当日危害、看见健康负债”为核心的戒烟辅助工具。产品不是医学诊断工具，不提供治疗建议，只用于行为记录、自我管理和风险提醒。

核心目标：

- 让用户知道今天已经取出/消耗了多少支。
- 让用户知道库存代表多少潜在健康负债。
- 让用户在取烟前后获得明确但不过度阻塞的风险提醒。
- 让用户用账本方式复盘行为，而不是只靠意志力。

## 2. 平台拆分

Flutter 版：

- Android + iOS 手机端主产品。
- 实现完整手机端功能：首页、资产、账本、我的、模型、设置、账户同步占位、帮助与关于、本地存储、通知、导出。
- 后续 Apple Watch / Wear OS 不强行复用 Flutter 手机端 UI，单独原生轻量扩展。

Harmony 版：

- HarmonyOS 手机端 / 华为生态版。
- ArkTS / ArkUI 原生实现。
- 复用同一产品规格、公式、文案原则和验收清单。
- 通知、权限、后台、设备联动按 HarmonyOS 原生能力实现。

## 3. 首版范围

必须包含：

- 首页
- 资产
- 账本
- 我的
- 模型与证据
- 设置
- 账户与同步占位
- 帮助与关于
- 添加资产
- 取出一支
- 放回库存/撤销一次
- 清空库存并记录退役
- 编辑戒烟目标
- 每日复盘弹窗
- 小 i 指标说明弹窗
- 重置本地数据二次确认
- 本地持久化
- CSV 导出
- 手机系统通知栏提醒

首版不包含：

- 真正云同步
- 医疗建议
- 社区功能
- 商城或购买推荐
- 手表端完整复杂功能

## 4. 信息架构

底部导航只保留四项：

- 首页
- 资产
- 账本
- 我的

二级入口：

- 我的 -> 模型与证据
- 我的 -> 设置
- 我的 -> 账户与同步
- 我的 -> 帮助与关于
- 我的 -> 戒烟目标
- 我的 -> 手表端说明或未来入口，不进入首版复杂实现

## 5. 首页规格

首页必须展示：

- 产品名：戒烟有数
- 肺部动态渲染/肺部负担视觉
- 当日危害
- 今日已取出支数
- 目标支数
- 库存支数
- 真实日耗
- 健康净值
- 健康负债
- 健康折算
- 主要动作：取出一支
- 次要动作：放回库存

UI 规则：

- 真实日耗、健康净值、健康负债样式统一，字体大小统一。
- 健康负债值不要换行。
- “实时渲染”和“每取出一支”的辅助文案不能挤压核心指标。
- 首页健康净值需要小 i 提示。
- 首页当日危害、首页健康负债如有理解成本，应提供小 i 说明。

## 6. 资产页规格

资产页必须展示：

- 香烟库存净值
- 健康负债
- 资产列表
- 默认取烟资产标识
- 风险标签，如偏高、偏低、低库存等
- 添加资产入口

默认标识规则：

- “默认取烟”标识必须比风险标签更小。
- 不得与“偏高/偏低”等风险标签抢同一视觉位置。
- 默认标识优先放在品牌标题附近的小标签位置。

添加资产表单规则：

- 品牌必填，建议不超过 16 字。
- 焦油量范围：1-20mg。
- 尼古丁范围：0.1-3mg。
- 库存范围：1-200 支整数。
- 整包价格范围：1-200 元。
- 表单提交无效时保持弹窗打开。
- 错误字段红色高亮。
- 字段下方显示错误文案。
- 底部显示错误摘要。
- 表单不是只依赖系统键盘类型，必须有业务校验。

## 7. 账本页规格

账本页必须展示：

- 今日流转
- 当日危害警示分
- 今日取出/放回/清空等事件
- 每条事件的品牌、时间、焦油、成本、健康折算、备注

布局规则：

- “当日危害警示分”和数值保持上下布局。
- 不要把“当日危害警示分 74”挤在同一行。
- 当日危害警示分需要说明入口。

## 8. 我的页规格

我的页必须包含：

- 戒烟目标
- 模型与证据
- 设置
- 账户与同步
- 帮助与关于
- 导出账本入口或跳转入口
- 手表端未来入口/说明

帮助与关于：

- 显示当前原型/App 版本。
- 必须有“检查更新”按钮。
- 可提交反馈。
- 检查更新按钮首版可模拟：显示“当前已是最新版本”。

## 9. 设置页规格

设置项：

- 严厉危害警示
- 取烟反馈
- 每日复盘提醒
- 隐私优先/本地优先
- 跨设备同步入口
- 导出戒烟账本
- 本地数据记录数
- 重置本地数据

严厉危害警示规则：

- 不阻止取烟。
- 达到干预线后，取烟提交成功，再触发手机系统通知栏三连提醒。
- APP 内不需要三连提示。
- 若通知权限未授权，记录仍成功，设置页显示权限状态和引导。

每日复盘提醒：

- 默认可设为 21:30。
- 系统通知提醒用户复盘。
- 打开 APP 后显示单个每日复盘弹窗。
- 每日复盘弹窗可跳转账本。

重置本地数据：

- 必须使用产品内二次确认弹窗。
- 不得使用浏览器 confirm 或平台默认生硬确认替代。
- 弹窗显示“高风险操作”。
- 弹窗显示当前本地记录数和库存支数。
- 弹窗列出影响：会恢复初始库存、会清空当前同步状态、不会影响已导出的文件。
- 点击取消：关闭弹窗并留在设置页，不清数据。
- 点击确认重置：恢复初始库存、账本、目标、设置和同步状态。

## 10. 取烟流程

取烟主流程：

1. 用户点击“取出一支”。
2. 打开取烟 Sheet。
3. 默认资产排第一。
4. 用户可选择资产、场景、备注。
5. 备注常驻显示，但不是必填。
6. 点击记录取出。
7. 扣减库存，写入账本，更新首页和模型指标。
8. 若严厉危害警示开启且达到干预线，提交成功后触发手机系统通知栏三连提醒。

取消规则：

- 用户关闭取烟 Sheet，不改变库存、不写账本。

放回规则：

- 放回库存用于撤销最近一次真实成本。
- 放回后库存 +1，今日已取出 -1 或记录相应 return 事件。
- 必须避免出现负数。

## 11. 指标与公式

所有指标必须显示“非医学诊断，仅用于行为反馈/自我管理”的合规提示。

今日包年增量：

- `todayPackYearIncrement = todayCigarettes / 20 / 365`

健康折算：

- `healthMinutes = cigarettes * 20`
- 页面可显示为分钟或小时。

库存潜在包年：

- `stockPotentialPackYears = stockCount / 20 / 365`

健康负债：

- 首页健康负债：可基于当日已取或当前上下文显示，必须在说明中写明口径。
- 库存健康负债：`stockHealthDebt = stockCount * 20` 分钟。

肺部负担指数：

- `lungScore` 是综合负担指数，用于健康净值计算。
- 具体分段可由产品内 QuitCalculator 固定实现。

健康净值：

- `netHealth = clamp(100 - lungScore, 0, 100)`
- 不得使用 `100 - dailyHarmScore`。

当日危害警示分：

- `dailyHarmScore` 表示当日行为风险。
- 用于当日警示，不等同于长期健康净值。

## 12. 模型与证据页

必须说明：

- 今日包年增量
- 健康折算
- 库存潜在包年
- 健康净值
- 当日危害警示分
- 健康负债
- 计算口径
- 公开模型参考与局限
- 非医学诊断提示

## 13. 小 i 说明弹窗

通用规则：

- 小 i 用于解释指标，不用于堆积长篇帮助。
- 关闭按钮 34x34。
- X 图标必须视觉居中。
- 移动端不得遮挡核心内容。
- 内容包含：指标含义、公式或口径、局限性、非医学诊断提示。

## 14. 通知规则

手机系统通知栏：

- 每日复盘提醒。
- 严厉危害警示三连提醒。

APP 内：

- 每日复盘只显示单个复盘弹窗。
- 严厉危害警示不做 APP 内三连提示。

三连提醒建议：

- 3 条本地通知。
- 间隔约 850ms，可由平台能力调整。
- 文案不羞辱用户，不做医疗恐吓。
- 目标是中断惯性行为。

## 15. 本地数据与导出

本地优先：

- 库存、账本、每日记录必须本地持久化。
- 目标、默认资产、设置开关必须持久化。
- 登录与云同步首版可占位。

导出：

- 支持 CSV。
- 包含时间、动作、品牌、焦油、成本、健康折算、备注。
- 已导出的文件不受重置本地数据影响。

## 16. 手表端未来规格

手表端不进入手机端首版实现。

未来手表端只保留：

- 看今日状态
- 快记一支
- 长按撤销
- 复盘入口
- 跳转手机端账本或同步状态

手表端禁止搬运：

- 资产复杂管理
- 模型与证据长文
- 设置复杂项
- 账户同步复杂流程

## 17. 验收与测试

移动端尺寸规则：

- `390x844` 是核心验收视口之一，不是实现时的固定尺寸。
- Harmony 版必须做响应式/自适应布局。
- 不得把主页面、手机壳、内容区固定写死为 `390x844`。
- 页面宽高应根据真实设备、安全区、状态栏、底部手势区自适应。
- 允许使用 ArkUI 自适应布局、百分比宽度、约束尺寸、滚动容器、系统安全区适配等方式实现。
- 内容高度超出时必须可滚动，底部主操作不能被系统手势区遮挡。
- 弹窗、Sheet、指标卡片、按钮应使用相对宽度和合理 max width，不写死单一设备宽度。
- 关键文字不能重叠或被截断；窄屏必要时允许说明文字换行。
- 首页健康负债这类核心数值要通过布局、字号、单位拆分等方式尽量保证不换行。
- 横屏不是首版重点，但不能白屏、崩溃或出现不可恢复遮挡。

移动端验收尺寸：

- `390x844` 作为对齐当前原型的基准验收尺寸。
- 至少一个小屏尺寸。
- 至少一个常规 Harmony/Android 手机尺寸。
- 至少一个大屏尺寸。
- 常见 iPhone 尺寸。
- 阶段报告必须写清楚本阶段验证过哪些尺寸。
- 如果某阶段暂时无法多尺寸验证，报告必须说明原因和补验计划。

每阶段必须验证：

- UI 验证
- 功能验证
- 逻辑/算法验证
- 弹窗与 Sheet 验证
- 设置状态验证

关键测试：

- 添加资产表单错误态。
- 默认资产标识不与风险标签冲突。
- 取烟备注常驻且选填。
- 取烟提交更新库存、账本、首页指标。
- 取消取烟不更新状态。
- 放回库存撤销一次真实成本。
- 严厉危害警示不阻止取烟，只发系统通知栏三连提醒。
- 每日复盘弹窗可跳转账本。
- 小 i 关闭按钮居中。
- 重置本地数据取消不清数据，确认才重置。
- 健康净值使用 `100 - lungScore`。

## 18. 阶段报告要求

每个阶段完成后必须生成报告，路径：
`docs/reports/phase_xxx_report.md`

报告必须包含：

- 阶段目标
- 已完成内容
- 新增或修改文件
- UI 验证方式与结果
- 功能验证方式与结果
- 验证命令与结果
- 未能验证的内容和原因
- 遗留问题
- 下一阶段注意事项

Harmony 版额外注意：

- 用 ArkTS / ArkUI 原生实现，不照搬 Flutter 代码结构。
- 保持同一产品规则和验收标准。
- 通知、权限、后台任务、分布式/设备联动后续按 HarmonyOS 原生能力评估。
- 如果 Harmony 手表端后续进入范围，仍应只做轻量功能，不搬运手机端复杂页。
